# 07_Business_Rules.md

# Business Rules

## DeRoyal Hotspot OS (DHOS)

**Version:** 1.0

**Status:** Phase One

---

# 1. Purpose

This document defines all business rules that govern how the application behaves.

These rules must be enforced by the backend and cannot be bypassed by the frontend.

---

# 2. Internet Plans

Each plan belongs to exactly one Bandwidth Profile.

A plan may have many vouchers.

A disabled plan cannot activate vouchers.

Deleting a plan is only allowed if no vouchers are associated with it.

Plans cannot have duplicate names.

Plans must have a duration greater than zero.

---

# 3. Bandwidth Profiles

Each profile represents a MikroTik Queue Profile.

Example

Bronze

Silver

Gold

Multiple plans may share the same profile.

Deleting a profile is only allowed if no plans reference it.

Profile names must be unique.

---

# 4. Voucher Generation

Voucher codes are generated by the system only.

Administrators cannot manually choose voucher codes.

Voucher codes must be unique.

Voucher generation supports bulk creation.

Maximum vouchers per request:

5000

Each generated voucher immediately belongs to a plan.

Every generated voucher begins with the status:

Unused

---

# 5. Voucher Code Format

Voucher codes

Uppercase

No spaces

No special characters

Randomly generated

Recommended length

10 to 12 characters

Example

```
A7FK9LQ2XZ
```

Codes must never be predictable.

---

# 6. Voucher Activation

A voucher can only be activated once.

A voucher must belong to an active plan.

A disabled voucher cannot activate.

An expired voucher cannot activate.

Activation creates a hotspot user on the MikroTik router.

Activation records:

Activation Time

IP Address

MAC Address

Router Username

Hotspot Session

After activation:

Voucher status changes to Active.

---

# 7. Voucher Expiration

The MikroTik router is responsible for enforcing session duration.

The application records the activation.

When the session expires:

Internet access is terminated.

Voucher status changes to Expired.

Voucher cannot be reused.

---

# 8. Administrator Accounts

Administrator email addresses must be unique.

Passwords must never be stored in plain text.

Passwords are hashed using Argon2.

Only active administrators may log in.

Inactive administrators cannot access the system.

---

# 9. Authentication

JWT access tokens expire after the configured duration.

Expired tokens require reauthentication.

Protected endpoints reject missing or invalid tokens.

Logout invalidates the active session.

---

# 10. Dashboard

Dashboard statistics are calculated from database records.

No values should be manually entered.

Dashboard cards update in near real time.

---

# 11. Router Connection

The system checks router connectivity before activation.

If the router cannot be reached:

Voucher activation stops immediately.

Voucher status remains Unused.

The customer receives an appropriate error message.

---

# 12. Hotspot Sessions

Each session belongs to exactly one voucher.

A session records:

Username

IP Address

MAC Address

Login Time

Logout Time

Session Status

Sessions are retained for reporting purposes.

---

# 13. Logging

The following actions are always logged.

Administrator Login

Administrator Logout

Voucher Generation

Voucher Activation

Voucher Deletion

Plan Creation

Plan Update

Plan Deletion

Router Connection Failure

Settings Update

Unexpected Errors

Logs cannot be edited.

---

# 14. Input Validation

Required fields cannot be empty.

Whitespace is trimmed.

Voucher codes are converted to uppercase.

Invalid characters are rejected.

Duplicate records are prevented.

---

# 15. Search Rules

Search is case insensitive.

Partial matching is supported.

Pagination is applied to large datasets.

Filters can be combined.

---

# 16. Deletion Rules

Unused vouchers may be deleted.

Activated vouchers cannot be deleted.

Plans referenced by vouchers cannot be deleted.

Bandwidth profiles referenced by plans cannot be deleted.

Administrators cannot delete their own account.

---

# 17. Export Rules

Voucher exports support:

PDF

CSV

Exports include:

Voucher Code

Plan

Bandwidth Profile

Status

Creation Date

Activation Date

---

# 18. Error Handling

All errors return standardized JSON responses.

System failures never expose internal server details.

Validation errors identify the offending field.

---

# 19. Security Rules

HTTPS is mandatory.

SQL injection protection is required.

Cross Site Scripting protection is required.

Rate limiting is enabled.

Sensitive configuration is stored in environment variables.

Passwords are never returned by any API.

---

# 20. Future Business Rules

Reserved for:

Paystack Payments

Customer Accounts

Recurring Plans

Resellers

Multiple Routers

Multiple Locations

Promotional Codes

Automatic Renewals

Bandwidth Quotas

Usage Tracking

---

# 21. Business Rule Success Criteria

The system is considered compliant when:

All voucher activations follow the defined validation process.

No voucher can be reused.

All administrator actions are recorded.

Plans and bandwidth profiles maintain referential integrity.

Router failures do not consume vouchers.

The backend consistently enforces every rule regardless of the frontend implementation.
